# Iteration 095 Executive Report

## Decision

Do not promote the event sleeve into the frozen production master.

## What We Learned

The split windows resolved the ambiguity left by `iter_094`:

- the sleeve is useful in `2025`
- mostly neutral in `2024`
- weaker than control in `2026 YTD`

That means it is a real but regime-sensitive sleeve, not an all-weather upgrade.

## Winner

Inside the positive window `2025`, the best row was `enterprise4 sleeve 10%`:

- `40.771%` return
- `Sharpe 2.112`
- `4.7%` drawdown

Inside the same positive window, `platform5 sleeve 15%` was nearly as strong:

- `40.467%` return
- `Sharpe 2.090`
- `4.4%` drawdown

## Main Risk

The sleeve loses its edge outside the positive window. It should not be treated as a production master replacement or default overlay.

## Recommendation

Keep the `IB` paper master frozen as-is. Reclassify the cloud event sleeve as a `positive-window shadow sleeve` and only revisit it through better regime detection, not through unconditional master promotion.
